## Permanent output architecture

### 1. Redo packet

- full passage or readable source image
- figures, tables, and answer choices
- timing and reasoning space
- no correct-answer markers
- no explanations

### 2. Learning guide

- every correct and incorrect question
- exact evidence location
- evidence-search method
- minimum sufficient information
- deduction chain
- question-specific distractor analysis
- explicit shortcut and why it works
- strategy success/failure
- timing diagnosis
- personal error links
- Anki, MCATalyst, Whimsical, and Goodnotes destinations
- Cornell-style writing space

### 3. Separate answer key

- letters only
- kept separate to avoid spoilers during redos

## Non-negotiable quality rules

- Passage appears once per question block, not once per question.
- Include all answer choices.
- Correct questions are learning material too.
- Do not write only “use proportionality,” “look at the graph,” “apply the equation,” or “use passage evidence.” Show the actual operation, comparison, substitution, or search path.
- Do not reuse one generic distractor paragraph across unrelated questions.
- CARS entries must identify the practical question type and exact passage evidence.
- A claim that a pattern is common must be supported by multiple real examples; do not convert one useful observation into an absolute rule.

## Resume checklist

Before continuing an exam guide:

1. Confirm the exact original source file is accessible in the current session.
2. Identify the exact exam, section, passage, and next question in `TASK_LEDGER.md`.
3. Verify the passage, figures, question, choices, and official explanation.
4. Write the question-specific learning entry manually.
5. Render and inspect the pilot or batch.
6. Record the last verified item and next exact item before stopping.

Do not infer source access from a filename or prior chat claim. Open the actual file in the active session.
