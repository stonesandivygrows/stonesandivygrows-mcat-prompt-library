# UWorld MCAT Science Practice-Test Autopsy Prompt

## Purpose

Use this prompt to convert a UWorld science practice set plus the student's spoken review into a diagnostic report that identifies content gaps, reasoning errors, timing problems, experiment-reading weaknesses, and durable repair actions.

This workflow applies to:

- Biology
- Biochemistry
- General Chemistry
- Organic Chemistry
- Physics/Math
- Psychology/Sociology

Do not include CARS unless explicitly requested. CARS should be reviewed separately as passage reasoning, author logic, strategy, and answer-choice analysis.

## Inputs

The student may provide any combination of:

1. UWorld practice-test PDF, export, screenshots, or copied text
2. UWorld explanations and educational objectives
3. Audio-review transcript
4. Scratch paper or handwritten work
5. Timing data and score report
6. Written reflections or notes
7. Kaplan-to-UWorld mapping references

Missing material should be reported, but the review should continue using the strongest available evidence.

## Source priority

Use sources in this order:

1. UWorld test/practice material supplied by the student
2. Student's audio-review transcript
3. Scratch paper and written notes
4. Kaplan chapter organization for content mapping
5. UWorld topic organization for targeted repair
6. MCAT Scientific Inquiry and Reasoning Skills

Scientific skill labels:

- Skill 1: Knowledge of Scientific Concepts and Principles
- Skill 2: Scientific Reasoning and Problem-Solving
- Skill 3: Reasoning About the Design and Execution of Research
- Skill 4: Data-Based and Statistical Reasoning
- Strategy/timing issue
- Scratch-paper issue
- Emotional/familiarity issue

## Core review rules

- Do not merely explain the correct answer.
- Diagnose what happened in the student's reasoning.
- Separate content failure from process failure.
- Translate unfamiliar jargon back into familiar MCAT concepts.
- Quote or closely reference the student's own reasoning when useful.
- Address every meaningful question, realization, correction, and note-to-self in the transcript.
- Never invent missing test wording, figures, reasoning, timing, or performance data.
- Mark inferred conclusions as inferred.
- Prefer depth over shallow completeness.
- When the report is too long, stop at a natural boundary, record exact completion status, and continue in later fragments.
- All fragments must append to one running downloadable document.
- Preserve a completion map so no question or passage is silently omitted.
- Do not reproduce full copyrighted question-bank content in GitHub summaries. Keep raw exports and full generated reports in private storage.

# Required Report Structure

## SECTION 1: Uploaded Material Inventory

Identify whether each item is present:

| Material | Present? | Notes |
|---|---:|---|
| UWorld questions | | |
| UWorld explanations | | |
| Passage text | | |
| Figures/tables/images | | |
| Selected answers | | |
| Correct answers | | |
| Timing data | | |
| Score data | | |
| Audio transcript | | |
| Scratch paper | | |
| Written notes | | |

Then state:

- what is missing
- how that limits the review
- which source best shows what happened
- which source best shows why it happened

## SECTION 2: Executive Diagnostic Summary

Report:

- overall performance pattern
- major strengths
- major weaknesses
- top 3–5 reasons for missed questions or lost time
- whether the dominant issue was content, passage decoding, question interpretation, mathematics, experiment interpretation, graph reading, elimination, pacing, anxiety, or familiarity
- highest-priority targets before the next set

Do not give generic advice. Tie every conclusion to evidence.

## SECTION 3: Audio Transcript Mining

### A. Questions asked out loud

List every meaningful question the student asked during review.

### B. Notes to self

Extract statements such as:

- I need to remember this
- I keep forgetting this
- I should do this next time
- I did not realize this
- I got tricked by this
- this wording confused me

### C. Reflections and feedback

Capture comments about:

- process
- timing
- confidence
- frustration
- confusion
- fear or unfamiliarity
- what became clear after review
- what the student learned

### D. Personal pattern clues

Organize into:

- Keep doing this
- Stop doing this
- Review this content
- Practice this skill
- Add to Anki
- Note to self for the next test

## SECTION 4: Test-Level Pattern Report

Create:

| Pattern | Evidence from test/transcript | MCAT skill type | Why it matters | How to fix it |
|---|---|---|---|---|

Distinguish among:

- missing knowledge
- knowledge available but not retrieved
- correct reasoning executed too slowly
- directionality reversal
- unsupported inference
- overreliance on background knowledge
- failure to use passage data
- experimental-variable confusion
- graph/axis/legend misreading
- arithmetic or equation setup error
- answer-choice wording trap
- late-set time collapse

## SECTION 5: Passage Autopsy

For every passage, use the following structure.

### Passage [number]: [topic]

### A. Big picture

- What the passage is really about
- Core science underneath
- Scary jargon translated
- Kaplan/UWorld topic connections
- What the MCAT was likely testing

### B. Passage translation

| Passage detail | Plain-English meaning | Why it matters | What it connects to | What not to overthink |
|---|---|---|---|---|

Include all answer-relevant:

- terms
- molecules
- enzymes
- structures
- organs
- cell types
- hormones
- equations
- numbers and units
- graph axes
- experimental conditions
- controls
- variables
- relationships
- trends
- comparisons
- conclusions

### C. What was scary but actually simple?

| Scary wording/detail | Familiar MCAT concept | Simple translation | How to recognize it next time |
|---|---|---|---|

### D. Experimental setup

When applicable, identify:

- research question
- hypothesis
- independent variable
- dependent variable
- control group
- experimental groups
- constants
- measured output
- supported conclusion
- unsupported conclusion
- result that would strengthen the claim
- result that would weaken the claim

### E. Figure/table/image breakdown

For every figure, table, pathway, blot, gel, molecule, or image:

- identify the axes, units, groups, legend, and comparison
- state the minimum conclusion supported
- state what the image does not establish
- explain how the figure connects to each related question
- distinguish passage data from outside expectations

### F. Minimum useful scratch paper

Give the shortest useful map for that passage. Prefer compact forms such as:

```text
IV:
DV:
Control:
Result:
Cause -> effect
Group A > Group B
```

Do not recommend full-sentence passage summaries during the timed phase.

### G. Passage-level coaching takeaway

State the single most important behavior to carry into the next comparable passage.

## SECTION 6: Question-by-Question Diagnostic Review

Review every question, including correct answers.

Use:

| Question | Result | What it tested | Student's reasoning | Why the correct answer works | Why the chosen answer fails | Error type | Repair rule |
|---|---:|---|---|---|---|---|---|

For each question:

1. Translate the stem into plain English.
2. Identify the tested concept or skill.
3. Reconstruct the student's reasoning from the transcript.
4. Explain the correct answer.
5. Explain the exact flaw in the selected answer.
6. Identify whether the error was content, reasoning, wording, directionality, data, mathematics, strategy, or time.
7. Give one concise repair rule.
8. State whether it deserves an Anki card, practice drill, reread, or no further action.

For correct questions, identify what should be preserved rather than creating unnecessary work.

## SECTION 7: Wrong-Answer Wording Autopsy

For each missed or uncertain answer choice, explain the wording bit by bit.

Use:

| Phrase in answer choice | Literal meaning | Hidden assumption | Why supported/unsupported | Better translation |
|---|---|---|---|---|

Pay special attention to:

- cause/effect reversals
- necessary vs sufficient language
- absolute wording
- correlation vs causation
- mechanism claims unsupported by outcome data
- comparison direction
- increase/decrease reversals
- sender vs receiver
- template vs product
- structure vs function

## SECTION 8: Content Repair Map

Create a prioritized content table:

| Topic | Evidence of gap | Kaplan chapter | UWorld section | Priority | Required action |
|---|---|---|---|---:|---|

Classify each topic as:

- true content gap
- rusty recall
- vocabulary gap
- directionality gap
- integration gap
- not a content problem

Avoid assigning full-chapter rereads when a short repair is sufficient.

## SECTION 9: Scientific Reasoning and Experiment Repair

Summarize recurring weaknesses involving:

- IV/DV identification
- controls
- antagonist/agonist logic
- knockout/rescue logic
- wild-type vs mutant comparisons
- necessity vs sufficiency
- supported vs unsupported conclusions
- figure-based inference
- confounds
- study design
- statistical interpretation

Provide reusable decision rules.

## SECTION 10: Mathematics and Equation Repair

For every calculation or relationship question:

- identify the governing equation
- define each variable and unit
- show proportional reasoning before arithmetic
- identify any algebra, sign, exponent, or unit error
- state the shortest test-day setup
- distinguish variables that share a symbol but have different meanings

Do not confuse unit compatibility with proof of causal relationships.

## SECTION 11: Scratch-Paper Audit

If scratch paper is supplied, evaluate it directly.

If it is absent, clearly label the analysis as inferred.

Report:

- useful notes
- unnecessary notes
- missing arrows or labels
- calculations that should have been written
- calculations that should have remained mental
- whether full sentences slowed the student
- a better template for the next set

## SECTION 12: Timing and Pacing Autopsy

Identify:

- early time sinks
- skipped passages
- return behavior
- late-set collapse
- questions answered too quickly
- questions held too long
- whether correct reasoning consumed excessive time

Create:

- checkpoints for the same set length
- a rule for leaving a question
- a rule for returning
- an emergency protocol for the final questions

Suggested controlled-guess sequence:

1. Label the tested concept.
2. Identify the direction or relationship.
3. Eliminate two choices.
4. Commit, flag, and move.

## SECTION 13: Emotional and Familiarity Autopsy

Identify moments where unfamiliar wording, jargon, fear, or frustration changed performance.

Separate:

- genuine knowledge gaps
- temporary retrieval failure
- passage intimidation
- overthinking
- productive self-correction

Give a short replacement script for the next test, such as:

> The name is unfamiliar; the relationship is probably familiar. Find the variable, direction, or function.

## SECTION 14: Anki Candidates

Create only high-value cards.

Group by:

- content facts
- directionality
- experimental logic
- equations/units
- recurring personal traps

Each card should be atomic and testable.

Avoid turning entire explanations into cards.

## SECTION 15: Reread and Practice Assignments

Recommend only the smallest effective intervention:

- exact Kaplan subsection
- exact UWorld lesson/topic
- five-minute drill
- figure-reading drill
- complementarity/directionality drill
- equation drill
- experimental setup drill

Label priority as high, medium, or low.

## SECTION 16: Next-Test Operating Rules

Produce a concise pre-test list containing:

- non-negotiable timing checkpoints
- scratch-paper rules
- what to do when terminology is unfamiliar
- what to do when the figure conflicts with expectations
- what to do when two choices remain
- what to do when behind on time

Keep this short enough to read immediately before practice.

## SECTION 17: Completion and Quality Audit

Before declaring completion, verify:

- every question is accounted for
- every passage is accounted for
- all discretes are accounted for
- all missed questions received full analysis
- correct questions with useful lessons were reviewed
- transcript questions were answered
- notes-to-self were captured
- figures/tables were addressed
- content mapping is present
- Anki candidates are deduplicated
- pacing rules are actionable
- no copyrighted full question-bank material was added to public/versioned summaries

Finish with:

```text
Completion status:
Questions reviewed: __ / __
Passages reviewed: __ / __
Discretes reviewed: __ / __
Transcript fully mined: Yes / No
Scratch paper reviewed: Yes / No / Not supplied
Required clarifications remaining:
Next continuation point, if incomplete:
```

# Running-Document Protocol

When the report is generated in fragments:

1. Start from the latest verified document.
2. Append rather than replace prior work.
3. Record the question and passage range completed in each fragment.
4. Preserve a backup before major reconstruction.
5. Verify the first old section and last newly added section after saving.
6. Do not label the document complete until the completion audit passes.
7. Once complete, remove `Running`, `Draft`, or `Pending` from the canonical title.

Recommended completed private filename:

```text
YYYY-MM-DD__UWorld_Test_[number]__Complete_Science_Autopsy.docx
```
