# Current CARS Testing Strategy

## Current diagnosis

The main issue is no longer basic passage comprehension. The recurring pattern is:

- the passage's overall logic is usually understood;
- points are lost when answer choices restate the author's claim indirectly, too broadly, too narrowly, or with unfamiliar wording;
- uncertainty often causes wording memory to disappear, so the better target is location confidence rather than perfect recall;
- broad or philosophical answer choices are sometimes chosen over narrower answers that more closely match the passage;
- under full-length pressure, passage-reading strategy may remain intact while question-answering strategy is forgotten;
- a frequent recent error is identifying a question type from one keyword before reading the full stem, then imposing the wrong task on the question;
- scope words such as **passage**, **paragraph**, **author**, and **argument** materially change the task and must be noticed before answer evaluation.

## Core rule

> Read for the author's machine. Track where ideas live. Skip clutter. Verify wording only when the location is known. Read the entire question stem before naming the question type, identify its scope, and answer exactly what was asked.

## Interface and visual-load setup

Before beginning a timed section:

- check whether the passage/question layout is visually comfortable;
- zoom out when permitted if the passage feels physically oversized or requires excessive scrolling;
- do not assume a practice-platform layout problem reflects a comprehension failure;
- keep the interface check brief so it reduces visual load without becoming a new ritual.

Working reminder:

> Zoom/layout first; then start.

## First read

The purpose of the first read is to build a location map, not memorize wording.

After each paragraph, identify its job in a short mental label:

- setup/problem;
- thesis/claim;
- explanation;
- evidence/example;
- counterpoint;
- consequence;
- conclusion.

The first paragraph is not automatically the thesis. It is often only setup. The last paragraph deserves extra attention because it frequently contains the final claim, consequence, or author recommendation.

### Passage-setting method

When formal labels are hard to remember, ask:

> Where would this argument naturally appear?

Examples:

- courtroom or debate room: competing claims, defense, rebuttal;
- YouTube video essay: defense, correction, reinterpretation;
- opinion piece: critique or advocacy;
- Socratic seminar: competing perspectives or conceptual revision;
- review or comment section: evaluation of a work, idea, or interpretation.

This method is a comprehension aid, not a substitute for identifying the author's final position.

### Causal reversal passages

When the passage argues that scholars had the order backward, reduce it to arrows.

Example pattern:

> Old view: A -> B -> C  
> New view: C -> pressure for B -> A

Do not leave the causal order as an unresolved question once the author has answered it.

## Optional answer-choice preview

A limited preview may be used when the first question is clearly a main-idea, purpose, or central-point question.

- Read the answer choices only to identify possible thesis lanes.
- Then read the passage normally.
- Do not stack this with automatic last-sentence scanning, first-sentence scanning, or a full section scan.
- Do not preview every question stem; that scatters attention.

Status: **use selectively, continue testing**.

## Controlled skipping

Use selective skipping to avoid getting trapped in parentheticals, long appositives, digressions, lists of names, and decorative examples.

### Usually safe to postpone

- short parentheticals;
- name lists;
- extra examples;
- long descriptive interruptions;
- decorative wording that does not change the sentence's logic.

### Never skip on purpose

- contrast words: but, however, yet, although;
- correction words: rather, instead, not X but Y;
- conclusion words: therefore, thus, hence, so;
- author judgment words: merely, only, dangerous, wise, foolish, unfortunately;
- definitions;
- the sentence after a quote;
- the final sentence of a paragraph;
- any sentence that explains what an example means.

Working phrase:

> Read the spine first. Return to the ribs only when asked.

## Rereading rule

Do not use a blanket rule of never going back.

Before rereading, ask:

1. What am I checking?
2. Where is it?

If both answers are known, return briefly. If either is unknown, do not wander through the passage. Use the passage map and eliminate.

### Go back for

- detail or retrieval questions;
- specific phrase or word questions;
- analogy questions when the mechanism must be recovered;
- function questions, limited first to the target sentence plus one sentence before and after;
- implication questions when the relevant paragraph is known.

### Usually do not go back for

- broad main-idea questions when the thesis is already clear;
- author-attitude questions when the author's position is already known;
- vague 50/50 uncertainty without a retrieval target;
- panic rereading.

## Question-stem operating system

Before looking at the answer choices, complete this sentence:

> This question is asking me to judge ______.

Then identify:

1. **Task/type:** main idea, function, implication, assumption, strengthen, weaken, application, evidence, significance, etc.
2. **Scope:** passage, paragraph, author, argument, quoted phrase, or new scenario.
3. **Direction:** most, least, except, not, strengthen, weaken, support, challenge.

The complete task is:

> Question type + scope + direction.

Do not classify from a single keyword before reading the full stem.

### Scope-word rules

- **passage**: use the whole passage argument or main idea;
- **paragraph**: begin locally and stay within that paragraph unless broader context is required;
- **author**: predict the author's view, tone, or reasoning;
- **argument**: focus on claim, evidence, dependency, and conclusion rather than topic details;
- **according to**: stay close to explicit text;
- **implies/suggests**: take the smallest supported step beyond the text;
- **if true/suppose**: treat the answer choices as new information and test their effect;
- **least/except/not**: reverse-task warning.

### Reading mode

- Passage: read primarily in the head.
- Question stem: read slowly or subvocalize when needed.
- Answer choices: compare directly against the locked task.

Working reminder:

> Stem first: judge what?

## Question-type rules

### Main idea

Ask:

> What is the author trying to prove by the end?

Do not choose an answer that only summarizes the introductory setup.

### Function

Do not ask what the sentence is about. Ask:

> What larger claim does this sentence, example, quote, or person support?

Function means job, not topic.

### Implication, suggests, or inference

Ask:

> What is the smallest supported step beyond the text?

Avoid the most dramatic or most philosophically interesting extension.

### Assumption

Ask:

> What must be true for the author's reasoning to work?

Move one step backward from the conclusion rather than adding a new claim.

For examples used as evidence, ask:

> What must be true for this example to actually prove the author's broader claim?

### Analogy or application

Identify the mechanism, then match the relationship rather than the subject matter.

Examples:

- object in recognizable space -> choose another object-in-setting relationship;
- study -> define -> control or attack -> preserve both knowledge and aggression;
- mind affects body -> choose a new case of cross-directional influence.

### Challenge or weaken

Translate the stem as:

> Pretend each answer is true. Which one makes the author's argument or rule less secure?

The phrase "if true" usually introduces new information; it does not ask which answer is already stated in the passage.

### Least important to the argument

Assume the answer choices may all be true passage claims. Ask:

> Could the thesis still work without this claim?

Separate background or discovery history from argument machinery.

### Least supported by evidence or explanation

Do not ask only which claim lacks hard proof. Rank the support surrounding each claim.

Support can include:

- direct evidence;
- explanation;
- analogy;
- comparison;
- contextual reason.

A weak analogy may still provide more support than a bare unexplained assertion.

### Reduce significance

First identify the exact source of significance. Then ask:

> Which new fact beats or removes the original example's special status on that same dimension?

If the passage emphasizes earliest, oldest, first, unique, or unprecedented, another example that is earlier or more direct may reduce the original's significance.

### New-scenario referent rule

When the stem says **suppose another**, **additional sites**, **a new study**, or similar language:

- label the original thing;
- label the new thing;
- confirm which thing each answer choice describes.

Working reminder:

> Suppose another = compare original with new thing.

### Roman numerals

Do not hold the logic only in prose. Mark each statement independently:

```text
I   ✓
II  ✓
III ✗
```

Then reread the stem direction and match the answer-choice grouping.

### Author would agree

Choose the answer that preserves the author's mechanism, not the answer that sounds nicest, wisest, or most familiar.

### Validity or most useful evidence

Test the central dependency of the argument. Do not attack a claim the author never made, especially an exaggerated universal claim such as "all diseases" or "always."

## 50/50 rule

When two answers remain, compare them in this order:

1. Which answer actually answers the full stem?
2. Which answer is closer to the author's actual claim?
3. Which answer is narrower?
4. Which answer adds fewer assumptions?
5. Which answer preserves the author's mechanism?
6. Which answer avoids changing possibility into certainty or a tendency into an absolute?

Recurring trap: upgrading the passage.

Examples:

- different -> more complex;
- poor prediction -> human intelligence is untrustworthy;
- do no harm -> never act;
- two-way interaction -> one treatment should always come first;
- rational study can support aggression -> understanding removes danger.

## Whiteboard

Keep the whiteboard minimal.

Write only:

- timing checkpoints already chosen before the section;
- skipped or return-to question numbers;
- a very short passage anchor only when the passage is unusually abstract;
- one compact question-reminder line if needed.

Suggested layout:

```text
CARS

Stem: type + scope | P / ¶ / A? | NOT? | RN ✓/✗
Skipped: __ __ __
```

Alternative shorter reminder:

```text
Q out loud -> judge what? -> answer THAT
```

Do not create a full written map for every passage. The map should usually stay mental.

## Timing and checking before leaving a passage

- A wordy first question may be skipped immediately and written on the whiteboard.
- When behind, it is better to finish one passage coherently than to rush through several passages without comprehension.
- Do not repeatedly check the clock inside every question.
- Do not reopen several non-retrieval questions at the end merely because they felt uncertain.
- Revisit only when there is a specific reason, such as locating a phrase, checking a stem inversion, or confirming a known passage reference.
- Before leaving a passage, consider a brief answer-selection check, especially for Roman numerals, reverse-worded stems, and questions that may have been misclicked.

Status of passage-exit check: **continue testing for time cost versus error prevention**.

## Do

- check zoom/layout before starting;
- read the title;
- distinguish setup from thesis;
- identify each paragraph's job;
- preserve contrast and conclusion language;
- read the entire stem before classifying it;
- identify task, scope, and direction;
- highlight or mentally tag passage, paragraph, author, argument, least, except, and not;
- translate awkward answer choices into plain English;
- reread with a target;
- write skipped question numbers;
- use arrows for causal reversals;
- use ✓/✗ for Roman numerals;
- use the author's exact direction and level of strength;
- prefer the boring, text-bound answer over the grand interpretation.

## Do not

- scan the entire test before beginning;
- rank passages by apparent difficulty before reading them;
- build a full written passage map for every passage;
- stack answer-choice preview, first/last-sentence scanning, and multiple prereading rituals;
- skip every comma, dash, or parenthetical automatically;
- classify a question from one keyword before reading the whole stem;
- assume Jack Westin, UWorld, and AAMC will use identical wording;
- answer a nearby familiar question instead of the exact question asked;
- assume uncertainty means the first answer was wrong;
- treat "generally" as an absolute;
- treat "might" as certainty;
- let one vivid example replace the passage thesis;
- choose an answer because it sounds morally attractive;
- demand perfect wording memory from the first read.

## Strategy status

### Keep

- paragraph-job mapping;
- controlled skipping;
- targeted rereading;
- question stem read before classification;
- task + scope + direction;
- passage/paragraph/author highlighting or mental tagging;
- arrow maps for causal reversal passages;
- ✓/✗ notation for Roman numerals;
- zoom/layout check;
- minimal whiteboard.

### Continue testing

- answer-choice preview for an early main-idea question;
- one-line passage map for every passage versus only abstract/causal passages;
- brief passage-exit answer check;
- physical highlighting of scope words versus mental tagging;
- subvocalizing every answer choice versus only confusing ones.

### Drop or limit

- full-section scanning before beginning;
- automatic last-sentence scanning before every passage;
- full written maps;
- question-type classification from isolated keywords;
- panic rereading;
- using one platform's wording as the template for every test source.

## Current benchmark

Recent practice continues to show that the passage thesis is often understood. The newest full-length review indicates that many lost points came from preventable question-processing errors: missing scope words, confusing new information with passage claims, losing the referent in hypothetical scenarios, and matching Roman numeral groupings incorrectly. The next priority is therefore not adding more passage-reading techniques. It is preserving the question-answering operating system under timed pressure.
