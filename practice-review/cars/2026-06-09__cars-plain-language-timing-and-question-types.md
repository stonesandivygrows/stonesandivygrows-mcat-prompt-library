# CARS Plain-Language Timing and Question-Type Guide

## Purpose

This file records the current CARS rules that are simple enough to use under test-day stress. It replaces complicated passage-by-passage subtraction, wall systems, and jargon-heavy labels with question-number checkpoints and plain-English question rules.

## Core timing system: use question-number checkpoints

Do not calculate how many minutes each passage should take during the section. Do not remember the exact start time of each passage.

Check the clock only at the major 10-minute marks:

| Time remaining | Around question |
|---:|---:|
| 80 | Q6 |
| 70 | Q12 |
| 60 | Q18 |
| 50 | Q24 |
| 40 | Q30 |
| 30 | Q36 |
| 20 | Q42 |
| 10 | Q48 |

### How to interpret the checkpoint

- Within about 1–2 questions: essentially on track.
- About 3–4 questions behind: reduce rereading and move faster.
- More than 4 questions behind: stop spending extra time on 50/50 questions; choose, flag, and move.
- Ahead: bank the time. Do not deliberately slow down.

## When behind

Use one rule:

> No luxury rereads.

A useful reread has a target:

- find a named detail;
- check a quoted phrase;
- verify a stem inversion such as NOT, EXCEPT, LEAST, or INCONSISTENT;
- confirm a known paragraph reference.

A luxury reread is rereading a large part of the passage because the answer feels uncomfortable.

## Question types in plain English

### Main idea / primary purpose

Ask:

> What is the author trying to convince me of by the end?

Choose the answer that covers the whole passage, not only the opening setup or one example.

### Detail / according to the passage

Ask:

> Where is the line?

These are the safest questions to revisit because the answer is searchable.

### Function / why mention this?

Ask:

> What job does this example, quote, person, or sentence perform?

Do not answer with what the example is about. Answer why the author included it.

### Suggests

Ask:

> What is the passage pointing toward?

This is a gentle inference. The answer may not be stated word-for-word, but it should remain close to the passage.

### Implies

Ask:

> What small unstated conclusion follows from this sentence or paragraph?

Take one small step beyond the text, not a dramatic leap.

### Assumes

Ask:

> What must be true for the author’s reasoning to work?

This is different from suggests or implies. It asks for the hidden bridge supporting the argument.

### Quick distinction

```text
SUGGESTS = what the passage points toward
IMPLIES = what follows one small step beyond the text
ASSUMES = what the argument needs to be true
```

### New information / suppose

Ask two questions:

1. What claim from the passage is being tested?
2. Does the new fact affect the same thing?

Label the new fact before choosing:

- count or prevalence;
- cause;
- outcome;
- evidence;
- mechanism;
- attitude;
- scope.

Do not treat two facts as connected merely because they share the same topic.

### Analogy / most like

Ask:

> What relationship must stay the same?

Ignore the topic first. Match the structure.

Example:

- direct instruction → constructivism
- being taught by someone → learning by doing

An answer about two different kinds of instruction would share the topic but miss the relationship.

### Vocabulary in context

Ask:

> How far does this word reach in this passage?

Abstract words are often narrower than their dictionary maximum.

Example:

- “universal” may mean everyone under similar conditions;
- it does not automatically mean every person in every possible situation.

### Evidence / strongest support

Ask:

> Which sentence actually proves the claim?

Do not choose a quote merely because it repeats the same nouns.

### Author would agree

Ask:

> Which answer preserves the author’s actual relationship and level of strength?

Do not choose the answer that sounds nicest, smartest, or most familiar.

## The main answer-choice trap

> Right topic, wrong job.

An answer can mention the correct subject while doing the wrong logical work.

Before reading the choices, finish this sentence:

> This question wants the answer that ______.

Examples:

- shows teacher-led learning becoming learning by doing;
- shows media simplifying a nuanced finding;
- affects recovery outcome rather than merely counting how many people exist;
- proves that a life story inspired a fictional work.

## Scope rule for abstract passages

When an answer uses a broad word, ask:

- for whom?
- under what conditions?
- compared with what?
- always, or only sometimes?

Common broad words:

- universal;
- valid;
- all;
- every;
- never;
- always;
- serious;
- important;
- necessary.

## Whiteboard role

The whiteboard is a dashboard, not a notebook.

Write only:

- question-number timing checkpoints;
- skipped or return-to question numbers;
- one symbol system if useful;
- one short reminder.

Suggested symbols:

```text
○ = 50/50
★ = findable detail to return to
/ = guessed or no idea
```

Suggested reminder:

```text
STEM → JOB → CHOICES
```

## Skipping rule

### Safe to return later

- concrete detail questions;
- line-reference questions;
- questions where the exact phrase is clearly searchable;
- NOT/EXCEPT/LEAST questions when the inversion needs checking.

### Usually answer before leaving the passage

- main idea;
- function;
- analogy;
- implication;
- assumption;
- author-agreement;
- reasoning 50/50 questions.

Once the passage map is gone, these questions become harder rather than easier.

## Strategies to stop using

- scanning the entire test before starting;
- calculating a custom leave time for every passage;
- remembering exactly when each passage began;
- using multiple timing systems at once;
- writing full passage maps;
- highlighting every contrast word;
- returning to old passages for vague reasoning uncertainty;
- choosing an answer because it contains familiar words from the passage.

## Final test-day sequence

1. Read the passage for the author’s point and paragraph jobs.
2. Read the question stem.
3. Translate the question into plain English.
4. State the answer’s required job.
5. Compare choices.
6. Check the clock only at the next major 10-minute checkpoint.
7. If behind, remove luxury rereads and keep moving.

## Current non-negotiables

- Question-number checkpoints, not passage subtraction.
- Stem first, then job, then choices.
- Suggests/implies are not the same as assumes.
- Match relationships, not topics.
- Define abstract words inside the passage’s limits.
- Return only when the target is known.
- Keep the system simple enough to remember under stress.
