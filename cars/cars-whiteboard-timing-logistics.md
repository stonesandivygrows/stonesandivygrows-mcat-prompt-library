# CARS Whiteboard Timing Logistics

## Purpose

Use the whiteboard only to prevent timing disorientation and forgotten return questions. It is not a passage-mapping or note-taking tool.

## Current default

For a short Jack Westin set such as **17 questions in about 30 minutes**, keep the board nearly blank.

```text
Return:
```

Write only the number of any question that must be revisited. Do not write explanations, answer choices, or passage summaries.

Use separate categories only when they genuinely help:

```text
S:
R:
```

- `S` = skipped questions that must be completed.
- `R` = answered questions worth reconsidering only if time remains.

A single `Return:` line is preferable when maintaining two categories creates extra friction.

## Observed calibration

On **June 9, 2026**, a 17-question Jack Westin set was completed in **29:44**. This supports using **about 30 minutes** as the practical time frame for a set of that size.

Pace alone does not prove that the timing method is optimal; accuracy and the number of rushed or abandoned questions still determine whether the pace was productive. The whiteboard should therefore preserve orientation without creating a second task.

## Passage-start timestamp

Recording the passage start time is useful because it restores time awareness without requiring constant clock monitoring.

Use a single compact mark:

```text
@29
```

This means the passage began when the countdown timer showed 29 minutes.

For later passages, add only the new countdown minute:

```text
@29  @19  @9
```

Do not write `P1`, `P2`, full timestamps, colons, or seconds unless passage identity would otherwise be unclear. The order of the marks already identifies the passage order.

### Timestamp rule

- Glance at the timer as the new passage opens.
- Round to the nearest whole minute.
- Write only `@` plus the minute.
- Return immediately to the screen.

The timestamp is worthwhile only if it takes about one second and prevents more timing confusion than it creates. Exact timing is less important than avoiding complete loss of time awareness.

## Strategy reference

A short printed or prewritten strategy list may remain visible, but it should be read only:

- before beginning;
- after noticing confusion or panic;
- after catching a repeated trap;
- before returning to skipped questions.

Do not reread the strategy list after every question.

A compact strategy cue may look like:

```text
QT → scope → MI
stuck = mark + move
```

Meaning:

- identify the question type;
- check the required scope: paragraph, passage, author, or argument;
- prefer the answer consistent with the main idea when the question is broad;
- when continued work is not producing new reasoning, mark and move.

## Do

- Record the passage start using one whole-minute countdown value.
- Write return-question numbers immediately.
- Separate true skips from lower-priority reconsiderations only when that distinction helps.
- Use the board when memory would otherwise fail.
- Keep attention on the passage interface rather than the board.

## Do not

- Do not write passage summaries, paragraph maps, tones, or main ideas by default.
- Do not copy answer choices or explain why a question was skipped.
- Do not calculate elapsed time repeatedly during the passage.
- Do not record seconds.
- Do not add checkpoints merely because a template recommends them if they create more monitoring work.
- Do not scan an entire multi-passage set merely to rank passages when that increases confusion.
- Do not treat a rigid 1:30 limit as automatic; move when additional time is no longer producing better reasoning.

## Minimal examples

### One passage or a short set

```text
@29
Return: 6 14
```

### When skip priority matters

```text
@29
S: 6 14
R: 9
```

### Three-passage set

```text
@29 @19 @9
Return: 6 14
```

## Decision rule

Every mark must answer one of two questions:

1. **When did this passage begin?**
2. **Which question must I revisit?**

If a mark answers neither question, it probably does not belong on the whiteboard.
