# CARS Timing, Whiteboard, and Skipping Protocol

## Purpose

This is the canonical CARS timing and whiteboard system. Use one global timing system, with one simple local safeguard inside each passage.

> **The whiteboard is a dashboard, not a notebook.**

## Core timing system

Use question-number checkpoints rather than calculating a separate leave time for every passage.

| Time remaining | Target question |
|---:|---:|
| 80 minutes | Around Q6 |
| 70 minutes | Around Q12 |
| 60 minutes | Around Q18 |
| 50 minutes | Around Q24 |
| 40 minutes | Around Q30 |
| 30 minutes | Around Q36 |
| 20 minutes | Around Q42 |
| 10 minutes | Around Q48 |

These are checkpoints, not exact deadlines.

### How to interpret the checkpoint

- **Within 1–2 questions:** On track.
- **About 3–4 questions behind:** Reduce rereading and stop polishing 50/50 decisions.
- **More than 4 questions behind:** Eliminate what you can, choose, flag, and move.
- **Ahead:** Keep the time. Do not deliberately slow down.

## Local passage safeguard: preserve one minute per question

This is the earlier per-passage rule:

> **Start answering once the timer leaves about one minute for every remaining question.**

For a roughly 10-minute passage block:

| Questions in passage | Begin questions by this elapsed minute at the latest | Equivalent time left in the passage block |
|---:|---:|---:|
| 5 questions | About **5:00 elapsed** | About **5:00 left** |
| 6 questions | About **4:00 elapsed** | About **6:00 left** |
| 7 questions | About **3:00 elapsed** | About **7:00 left** |

Plain-English version:

- **5Q passage:** do not still be reading after about minute 5.
- **6Q passage:** do not still be reading after about minute 4.
- **7Q passage:** do not still be reading after about minute 3.

This is not a command to stop reading mid-sentence. It is a maximum-reading warning. Finish the sentence or paragraph, state the passage direction, and begin the questions.

### Question-by-question countdown version

Once questions begin, the remaining passage time can act as a simple check:

- 7 minutes left → should be starting a 7-question set
- 6 minutes left → should be around the next question
- 5 minutes left → should have no more than about 5 questions remaining
- continue at roughly one minute per remaining question

Do not write this whole countdown on the whiteboard. Use it only when a passage read is running long.

### Why this does not conflict with the global checkpoints

- **Global checkpoints** tell you whether the whole section is on pace.
- **The one-minute-per-question rule** prevents one passage read from consuming the time needed to answer its questions.

The local rule is a rescue guardrail, not a second full timing system.

## Whiteboard setup

Write this before the section begins:

```text
CARS

80 Q6
70 Q12
60 Q18
50 Q24
40 Q30
30 Q36
20 Q42
10 Q48

Return: __ __ __

STEM → JOB → CHOICES
```

Optional symbols:

```text
○ = 50/50
★ = findable detail
/ = guessed or no clear path
```

Use only one symbol system. Do not create new codes during the section.

## What belongs on the whiteboard

Write only:

- the fixed timing checkpoints
- question numbers to revisit
- one very short passage anchor if a passage is unusually abstract
- one compact reminder such as **STEM → JOB → CHOICES**

## What does not belong on the whiteboard

Do not write:

- full paragraph summaries
- detailed passage maps
- every strategy you know
- full answer-choice comparisons
- custom passage-by-passage subtraction
- every passage start time
- every uncertain question

Writing should reduce confusion, not create another task.

## Clock-check rule

Check the clock only at the major checkpoints or after finishing a passage when needed.

Do not check:

- after every question
- repeatedly during one 50/50
- every time a sentence feels difficult

A clock check should answer only:

> **Am I generally on track, slightly behind, or meaningfully behind?**

## When behind

Use one rule:

> **No luxury rereads.**

### Targeted reread

A targeted reread has a specific goal and location:

- find a named detail
- check a quoted phrase
- verify **NOT / EXCEPT / LEAST / INCONSISTENT**
- confirm a known paragraph reference
- recover an analogy mechanism

### Luxury reread

A luxury reread means reopening a large part of the passage because the answer feels uncomfortable.

When behind, remove luxury rereads first. Do not remove careful stem reading.

## Skipping protocol

### Step 1 — Identify whether the question is returnable

Questions that are usually safe to return to:

- concrete detail questions
- line-reference questions
- vocabulary questions with a known location
- questions where the exact phrase is clearly searchable
- reverse-worded questions that need a final inversion check

Questions that usually become harder after leaving the passage:

- main idea
- function or purpose
- analogy
- implication or inference
- assumption
- author agreement
- broad reasoning 50/50 questions

Try to answer the second group before leaving the passage because the passage map is still active.

### Step 2 — Use the focused-attempt rule

If no clear path appears after a focused attempt:

1. reread the stem once
2. identify the question type and required job
3. eliminate what clearly fails
4. select the best remaining answer
5. flag only if returning could realistically help
6. record the question number
7. move

Do not flag a question merely because it felt difficult.

## What to flag

Flag when there is a concrete future action:

- retrieve a specific line
- verify a negative stem
- compare two exact phrases
- finish a known calculation of logic
- correct a suspected misclick

Do not flag for:

- vague discomfort
- “I might be wrong”
- wanting to rethink the whole passage
- every 50/50

## Passage-exit check

Before leaving a passage, perform a brief check only when time permits:

- Did I answer every question?
- Did I misread a reverse-worded stem?
- Did I misclick a Roman-numeral combination?
- Is there one flagged question with a specific retrieval target?

Do not reopen the entire passage set simply because some answers felt uncertain.

## Timing recovery levels

### On track

Continue normally.

- Read the passage for paragraph jobs.
- Use targeted rereading.
- Resolve reasoning questions while the passage map is active.

### Slightly behind

Remove low-value behaviors:

- repeated answer rereading
- overexplaining a 50/50
- rereading without a target
- writing extra notes

Keep:

- careful stem reading
- scope checks
- negative-word checks

### Meaningfully behind

Use the emergency sequence:

1. lock the stem task
2. use the passage main idea or local paragraph
3. eliminate contradictions, scope upgrades, and wrong relationships
4. choose
5. move

Do not sacrifice several later questions to rescue one current question.

## Timing strategies to stop using

- scanning the entire section before beginning
- ranking all passages by apparent difficulty
- calculating a custom leave time for each passage
- recording every passage start time
- using multiple global timing systems
- checking the clock every question
- saving many reasoning questions for the end
- leaving answers blank

## Timing strategies to keep

- question-number checkpoints
- one-minute-per-remaining-question passage safeguard
- minimal return list
- targeted rereading
- specific flag reasons
- brief passage-exit check
- preserving time when ahead
- reducing luxury rereads when behind

## Current whiteboard rule

> **Write the checkpoints once. Record only questions with a real return plan. Keep the passage map in your head.**
