# CARS Operating System and Decision Tree

## Purpose

Use this as the live play-by-play plan for a CARS passage. The goal is not to remember every strategy at once. The goal is to select the correct tool for the exact problem in front of you.

> **One task at a time. One strategy at a time. Do not stack tools unless the first tool fails.**

## Part 1 — Passage start

### Step 1 — Reset

Before opening the passage:

- Release the previous passage.
- Keep the mouth closed to prevent accidental subvocal reading.
- Use light finger tapping only if it helps track the line without slowing reading.
- Do not scan the full section or reorder all passages. That creates overload.

### Step 2 — Check the passage size

Notice the number of questions attached to the passage.

Use it only as a timing cue. Do not build a detailed timing calculation during the passage.

### Step 3 — Read the title and source

Take only a quick clue:

- topic
- field
- possible tone

Do not deeply interpret the title. Do not memorize the source.

### Step 4 — Read paragraph 1 carefully

Ask:

- What issue is being introduced?
- Who is speaking?
- What position appears to be on the table?

### Step 5 — Read each remaining paragraph for its job

After each paragraph, keep one short mental label:

- claim
- contrast
- cause
- example
- history
- objection
- response
- conclusion

Ask **what the paragraph is doing**, not how many details it contains.

### Step 6 — Handle confusion without stopping the passage

If a paragraph feels unclear, do not immediately reread everything.

Ask:

1. Who is speaking?
2. Is the tone positive, negative, or neutral?
3. Is this an old view, new view, example, objection, or response?
4. Does the author agree?

If still unclear, assign a rough role such as **setup**, **example**, or **criticism** and continue. Later paragraphs often clarify earlier ones.

### Step 7 — Finish with the passage direction

Before the first question, state the passage in one sentence:

> **The author uses ___ to argue/show/criticize ___ because ___.**

Do not write a detailed map unless the passage genuinely requires one.

## Part 2 — Question start

### Step 8 — Read the stem and identify the task

Highlight mentally or visually:

- **author**
- **passage**
- **paragraph**
- **except / least / inconsistent**
- **suggests / implies / assumes**
- **strengthen / weaken**
- **purpose / function**

Then translate the stem into plain English before looking for an answer.

### Step 9 — Do you understand the question?

#### Yes

Predict the job of the answer in a few words, then compare choices.

#### No

Use the stem-repair sequence:

1. Identify the subject of the question.
2. Identify the verb: asks, supports, weakens, implies, assumes, explains.
3. Identify the target: author, passage, paragraph, quoted phrase, or new scenario.
4. Restate the task in plain English.
5. Only then inspect the choices.

If still unclear after this sequence, flag, make the best supported choice, and move.

## Part 3 — Strategy selector

| Question type | Use this strategy | Do not use this strategy |
|---|---|---|
| Main idea | Use passage-wide direction, author payoff, and paragraph roles | Do not hunt for one impressive detail or quote |
| Detail | Return to the exact line and nearby context | Do not answer from broad theme alone |
| Function / purpose | Ask why the author included it **here** | Do not merely paraphrase what the sentence says |
| Suggests / implies / inference | Move one supported step forward | Do not use the assumption negation test or make a large philosophical leap |
| Assumption | Find the hidden bridge the reasoning needs; negate it when useful | Do not ask only what follows from the passage |
| Analogy | Reduce the original relationship to a mechanism and match it | Do not match topic words, setting, or surface similarity |
| Strengthen / weaken | Identify the exact claim or link, then test whether the choice helps or hurts it | Do not attack a nearby example that the argument does not depend on |
| Author agreement | Match the author's stance and passage direction | Do not pick an answer merely because it mentions the same topic |
| Vocabulary in context | Substitute each choice into the sentence and preserve the local meaning | Do not rely on the dictionary meaning alone |
| New information / application | Apply the passage rule to the new situation | Do not solve using outside knowledge |
| EXCEPT / least / inconsistent | Reverse the task before judging choices | Do not forget the negative word midway through the question |

## Part 4 — Answer-choice comparison

### Step 10 — Compare the relationship, not the topic

Before choosing, say the tested relationship in a few words:

- physical action affects mental result
- religion shapes historical knowledge
- example supports criticism
- same condition, same rule

An answer can mention the same topic and still perform the wrong job.

### Step 11 — Eliminate using four checks

1. **Task:** Does it answer the exact question?
2. **Support:** Can the passage support it?
3. **Scope:** Is it stronger or broader than the passage?
4. **Relationship:** Does it preserve the correct direction and mechanism?

### Step 12 — If stuck between two

Ask in this order:

1. Which answer requires fewer added assumptions?
2. Which answer matches the author's exact tone?
3. Which answer preserves the relationship rather than just the subject?
4. Does one choice change **can** into **should**, **some** into **all**, or effect into intention?

### Step 13 — Answer-change rule

Change an answer only for a concrete reason:

- missed negative word
- stem misread
- scope mismatch
- direction reversed
- direct contradiction
- clearer mechanism match

> **Discomfort alone is not evidence.**

## Part 5 — Confusion recovery

### If the passage sentence is unreadable

1. Find the subject.
2. Find the main verb.
3. Remove the interruption temporarily.
4. Preserve logical hinges such as **but, only, not, however, because, although**.
5. Reinsert the modifier only if it changes the claim.

### If all answers look wrong

- Re-read the stem, not the whole passage.
- Identify the task word.
- Choose the answer that is most passage-grounded and least dependent on extra assumptions.

### If the question feels overwhelming

- Do not open multiple strategies at once.
- Select one question type.
- Use that type's procedure.
- If no clear path emerges, flag, choose, and continue.

### If time is running short

- Use the passage thesis.
- Eliminate contradictions and scope upgrades.
- Choose the best-supported answer.
- Leave nothing blank.

## Part 6 — Timing and whiteboard rules

The whiteboard is a dashboard, not a notebook.

Record only:

- passage start time when useful
- question numbers to revisit
- minimal timing anchors

Do not write paragraph summaries, full strategy lists, or detailed calculations during the section.

### Skip rule

If there is no clear path after a focused attempt:

1. eliminate what you can
2. choose the best remaining answer
3. flag it
4. move

Do not let one 50/50 consume the next question's time.

## Operational summary

> **Read for jobs. Translate the stem. Select one strategy. Match the relationship. Control scope. Move when the path is gone.**
