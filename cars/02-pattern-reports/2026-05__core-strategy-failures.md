# May 2026 CARS Pattern Report — Core Strategy Failures

## Scope

This report consolidates recurring failures observed across the May Jack Westin review block and related April–May reflections. It is a cross-passage report, not an answer key.

## Pattern 1 — Topic Familiarity Over Passage Function

### Pattern

An answer can feel more natural or familiar while performing the wrong argumentative job.

### Evidence

- Preference for an answer using familiar mental-language wording even when another answer better preserved the passage’s body-to-mind relationship.
- Attraction to answer choices that shared passage vocabulary but did not reproduce the passage mechanism.

### Fix Rule

**Choose the answer that preserves the passage relationship, not the answer whose topic sounds closest.**

### Trigger

Two remaining answers both sound plausible, but one reproduces the author’s causal or comparative structure more exactly.

### Status

**Active risk — improving.**

## Pattern 2 — Retrieval Mode Used on Non-Retrieval Questions

### Pattern

Inference, suggestion, assumption, function, and main-idea questions were sometimes treated as line-matching tasks.

### Why It Fails

Exact wording can create false confidence when the question asks for a broader implication or argumentative role.

### Fix Rule

Route by stem:

- according to / states / mentions → retrieve
- implies / suggests → one supported step
- assumes → identify the missing bridge
- function / role → identify what the detail does
- author likely agrees / main idea → preserve the full thesis relationship

### Trigger

The stem contains **suggests, implies, assumes, role, function, validity, most likely agree**, or similar task words.

### Status

**Active risk.**

## Pattern 3 — Syntax Interruptions Consume Working Memory

### Pattern

Comma-heavy sentences, appositives, em dashes, titles, names, and inserted examples can obscure the main clause before the claim is identified.

### Why It Fails

The sentence is held as one undifferentiated block instead of being reduced to subject, verb, and point.

### Fix Rule

Use:

**Subject → Verb → Point**

Temporarily wrap removable interruption material in backticks. Restore it only if it changes the claim or a question asks about it.

### Trigger

Internal reaction: “I understand every word, but I do not know what the sentence is saying.”

### Status

**Modify and retest using the 3-layer system.**

## Pattern 4 — Passage Focus Identified Too Late

### Pattern

A passage may appear to discuss a topic or technique while its actual purpose is to explain why that topic or technique matters.

### Why It Fails

The reader tracks subject matter but not the author’s argumentative purpose.

### Fix Rule

After paragraph 2 or 3, ask:

**Is the author describing the thing, evaluating the thing, or explaining why the thing works?**

### Trigger

The passage repeatedly returns to effects, significance, success, failure, or consequences.

### Status

**Active risk.**

## Pattern 5 — Relationship Direction Gets Narrowed

### Pattern

A bidirectional claim can be remembered as only one direction because the passage gives more examples in that direction.

### Why It Fails

Example frequency is mistaken for thesis scope.

### Fix Rule

Before choosing, restate the thesis with arrows:

```text
A → B
B → A
or
A ↔ B
```

Then reject answers that preserve only half of the relationship when the thesis is reciprocal.

### Trigger

The passage uses language such as **vice versa, mutual, reciprocal, both, neither takes priority**, or equivalent wording.

### Status

**Active risk — explicit arrow check added.**

## Pattern 6 — Silent Skipping Creates Anxiety Debt

### Pattern

When a difficult question is skipped without a clear reason tag, working memory continues trying to track it.

### Why It Fails

Unfinished questions become invisible obligations, increasing time uncertainty and reducing focus on the current question.

### Fix Rule

When skipping, assign one brief category:

- stem unclear
- two-answer split
- retrieval needed
- time stop

Use navigation rather than memory to return.

### Trigger

The question exceeds the time boundary or produces repeated rereading without a new inference.

### Status

**Modify — navigation required.**

## Pattern 7 — Over-Scanning Before the Test Increases Overload

### Pattern

Scanning the entire test to select passage order introduced more executive demand than benefit.

### Fix Rule

Read passages in order unless deliberately running a controlled passage-order experiment. Check only the question count for the current passage to set pace.

### Trigger

Pre-passage planning begins taking longer than the passage itself should require.

### Status

**Stop for standard practice.**

## Pattern 8 — Temporary Confusion Leads to Restarting

### Pattern

A difficult opening paragraph can trigger immediate rereading before later context has a chance to clarify it.

### Fix Rule

Continue unless the core subject and stance are completely unavailable. Assign a temporary function label such as **setup**, **old view**, **example**, or **counterpoint** and move forward.

### Trigger

Reaction after one paragraph: “Wait, what?”

### Status

**Modify — keep moving, then reassess.**

## Pattern 9 — Reading Aloud Slows Processing

### Pattern

Mouthing or quietly reading aloud reduces speed compared with silent reading.

### Fix Rule

Keep the mouth relaxed and closed. Finger anchoring or light tapping may be used only if it improves visual tracking without imposing a slower rhythm.

### Trigger

Reading pace begins matching speech pace.

### Status

**Keep silent reading; retest finger anchoring.**

## Highest-Priority Rules

1. **Classify the question before choosing a strategy.**
2. **Preserve the passage relationship, not just its topic.**
3. **Reduce dense syntax to subject → verb → point.**
4. **Do not use retrieval as the default.**
5. **Track skipped questions through navigation, not memory.**

## Strategy Status Summary

### Keep

- Reading in order
- Title/source for light orientation
- Silent reading
- Paragraph-function awareness
- Comparing answer verbs, scope, and relationship direction

### Modify

- Last-sentence attention: use as a clue, not an automatic thesis rule
- Finger tapping: only if it does not slow pace
- Passage return: use when task requires local evidence
- Syntax removal: functional removal, not “skip commas”

### Stop

- Full-test scanning when it creates overload
- Written paragraph maps during timed work
- Treating every question as retrieval
- Choosing by familiar wording
- Restarting a passage immediately after temporary confusion

### Retest

- Three-layer syntax marking
- Brief skip-reason tags
- Mid-passage focus check after paragraph 2 or 3
