# CARS Strategy Operating System

## Purpose

This file consolidates the current CARS reading, question-answering, timing, interface, and review strategies into one durable reference. It distinguishes strategies to keep, strategies still being tested, and strategies to drop or limit.

## Core operating system

> Preview only if useful → read for thesis → identify stem type + scope → answer exactly that question.

Compressed version:

> Read for argument. Answer the stem.

## Interface and setup

- Zoom out before starting if the passage feels visually large or overwhelming.
- Check whether the passage and question layout is comfortable before beginning timed work.
- Treat poor interface layout as a testing-condition variable, not automatically as a comprehension failure.
- Do not scan the entire section before beginning; this has been overwhelming and not worth the cognitive cost.
- Keep the whiteboard minimal and limited to timing, returns/skips, and a small number of strategy triggers.

## Passage-reading strategy

- Read the passage mostly in the head rather than aloud.
- Use finger tapping or another unobtrusive pacing aid if it improves word tracking.
- Ask what the author is doing, not merely what topic is being discussed.
- Use the preferred “setting” frame when useful:
  - courtroom
  - debate room
  - Socratic seminar
  - YouTube video essay
  - opinion piece
  - review or comment section
- Translate formal passage types into intuitive versions:
  - correction: “Here is the misconception I am correcting.”
  - defense: “Here is why this deserves defending.”
  - critique: “Here is what is wrong with this view.”
  - revision: “I partly agree, but I need to modify it.”
  - explanation: “Here is why this phenomenon happens.”
  - debate: “Here are the competing sides and where I land.”
  - causal reversal: “People thought A caused B, but the passage argues B may have caused A.”
- For causal passages, compress the argument into arrows.
- Avoid stacking too many orientation steps. Do not automatically combine answer-choice preview, first-sentence reading, last-sentence reading, and paragraph scanning.

## Main-idea preview

- Preview answer choices only when an early question clearly asks for main idea, central point, or purpose.
- Use the answer choices as possible thesis lanes, not as conclusions to impose on the passage.
- Do not preview every question set automatically.

## Question-stem discipline

The main full-length lesson is that passage understanding can be correct while the wrong task is answered.

Before considering answers, finish:

> This question is asking me to judge ______.

Then identify both:

1. question type
2. scope

Question type alone is incomplete.

### Scope words

- passage: use the whole passage argument or main idea.
- paragraph: stay local unless broader context is needed.
- author: predict the author’s view, tone, or reasoning.
- argument: focus on claim, evidence, and conclusion.
- according to: remain close to explicit text.
- implies or suggests: make the smallest supported inference.
- if true or suppose: treat the information as new and test its effect.
- least, except, or not: reverse-task warning.

### Highlighting rule

Highlight or mentally tag:

- passage
- paragraph
- author
- argument
- least
- except
- not
- strengthen
- weaken
- assumption
- implies
- suggests

Do not categorize the question from one keyword before reading the complete stem.

## Answer-choice rules

### Exact-task rule

- Make sure the selected answer answers the exact question, not a nearby question.
- Read the stem and answer choice together when wording is confusing.
- Subvocalize the question stem and difficult answer choices if doing so clarifies syntax.

### Least important

Ask:

> Could the passage’s thesis still work without this claim?

This distinguishes background or scene-setting from argument machinery.

### Least supported by evidence or explanation

Count both:

- direct evidence
- reasons
- analogy
- comparison
- contextual explanation

A weak analogy can still count as more support than a bare assertion.

### Strengthen and weaken

- First identify the exact claim being affected.
- “If true” usually introduces new information.
- Relevance alone is insufficient; the new information must increase or decrease confidence in the claim.

### Assumption

Ask:

> What must be true for this example or premise to support the conclusion?

Also identify the scope: passage argument, paragraph argument, or author claim.

### Suppose another / new scenario

- Label which thing the answer choices describe.
- “Suppose another…” usually means compare the original with a new example.
- Track pronouns and referents before evaluating content.

### Roman numerals

Do not retain the evaluation only as prose.

Use:

- I ✓
- II ✓
- III ✗

Then reread whether the question asks for what is supported or what is not supported before matching the combination.

## Timing and passage transitions

- Use precomputed checkpoints rather than performing arithmetic under stress.
- Keep timing notes minimal.
- Before leaving a passage set, briefly check:
  - reverse words such as NOT or LEAST
  - Roman numeral combination
  - flagged or uncertain questions
  - possible misclicks
- Do not reread every question automatically if the section is already behind schedule.
- If a question feels incomprehensible, reread the stem before rereading the entire passage.

## Compact whiteboard trigger

> Zoom/layout | Stem: type + scope | P / ¶ / Author? | NOT / LEAST? | RN ✓✗ | Suppose = new thing

A shorter alternative:

> Q out loud → judge what? → match answer

## Strategies to keep

- Zoom/layout check before starting.
- Passage reading primarily in the head.
- Finger tapping if it aids tracking.
- Read for thesis and author function.
- Use intuitive passage-setting categories.
- Use arrows for causal order.
- Read the complete stem before naming the question type.
- Identify scope words.
- Make the answer respond to the exact task.
- Use checkmarks and Xs for Roman numerals.
- Track the original thing versus the new thing in scenario questions.
- Briefly verify risky answers before leaving a passage.

## Strategies still being tested

- Previewing answer choices before the passage only when Q1 is main-idea based.
- Writing a one-line map after every passage versus only difficult passages.
- Performing a mini-check before every passage transition.
- Physically highlighting scope words versus mentally tagging them.
- Subvocalizing every answer choice versus only confusing choices.

## Strategies to drop or limit

- Scanning the entire section before beginning.
- Reading the last sentence, first sentence, answer choices, and passage as a mandatory sequence.
- Writing full passage maps.
- Classifying a question from a keyword before reading the complete stem.
- Returning to the passage for every question regardless of task type.
- Treating Jack Westin, UWorld, or AAMC wording as fixed or universal.
- Treating “least supported” as meaning only “least direct evidence.”

## Full-length review lesson

The principal error pattern from the reviewed passage was not failure to understand the passage thesis. It was loss of question-stem discipline under pressure. The review process should continue distinguishing:

- passage comprehension errors
- task-identification errors
- scope errors
- referent errors
- answer-transfer or misclick errors
- timing/interface effects

This distinction prevents a low raw score from being misdiagnosed as a global reading-comprehension problem.
