# Current CARS Testing Strategy

Last updated: 2026-06-08

This is the current, simplified CARS system. It replaces older versions that depended on scanning the whole test, doing passage-by-passage arithmetic, counting timer “walls,” or writing full passage maps.

## 1. Core principle

Read for the author’s job, then answer the exact job in the question stem.

Before looking at answer choices, complete this sentence:

> This question wants the answer that ______.

Examples:

- shows what the author is arguing
- explains why an example was included
- applies the same relationship to a new situation
- identifies what must be true for the argument to work
- gives the narrowest conclusion supported by the passage

The biggest recurring trap is choosing an answer that mentions the right topic but performs the wrong job.

## 2. Reading strategy

### Keep

- Start passages in order unless the first paragraph is clearly unreadable and skipping will not create tracking problems.
- Read the title.
- Read paragraph 1 carefully enough to identify the issue or debate.
- After each paragraph, ask only: “What did this paragraph do?”
- At the end, state one sentence: “The author wants me to see X as Y.”
- Highlight only thesis statements, contrast turns, or one visual anchor when needed.

### Do not use

- Do not scan the entire test before beginning.
- Do not rank every passage by interest or difficulty.
- Do not write full passage maps during timed practice.
- Do not highlight every last sentence or every contrast word.
- Do not mouth or read the passage aloud; this was personally slower than silent reading.
- Do not add a new strategy in the middle of a timed set.

### Dense sentence repair

Only use this when a sentence is directly needed for a question:

1. Find the subject.
2. Find the main verb.
3. Ignore the comma/dash interruption temporarily.
4. Restate the sentence in plain English.

Do not perform full grammar surgery on every sentence during the first read.

## 3. Question-type rules in plain English

### Suggests

Meaning: What is the passage pointing toward?

Rule: Choose the answer most supported by the passage, even if it is not stated word-for-word.

### Implies

Meaning: What unstated idea reasonably follows from the text?

Rule: Take one small step beyond the sentence, not a large leap.

### Assumes

Meaning: What does the author need to be true for the argument to work?

Rule: Ask, “If this were false, would the argument become weaker?”

### Function

Meaning: Why did the author include this example, quote, person, or sentence?

Rule: Answer the author’s purpose, not the topic of the example.

### Analogy / most like

Meaning: Which new situation has the same relationship?

Rule: Ignore the topic first. Match the structure.

### New information / suppose

Meaning: Does the added fact support, weaken, or fail to affect the same claim?

Rule: Label the new fact before choosing:

- cause
- outcome
- prevalence/count
- mechanism
- evidence
- attitude
- scope

If the new fact changes a different variable, it may be irrelevant.

### Vocabulary in context

Meaning: What does this word mean here?

Rule: Use the sentence and paragraph, not the biggest dictionary meaning.

### Main idea / primary purpose

Meaning: What is the author trying to make the reader believe or see differently?

Rule: Choose the answer broad enough for the whole passage but narrow enough to preserve the author’s exact claim.

## 4. Two recurring error rules

### Do not pick by topic. Pick by job.

An answer can use the same nouns as the passage and still be wrong.

Before choices, identify the relationship:

- teacher-led learning → learning by doing
- prevalence/count → not the same as recovery outcome
- media headline → oversimplifies a nuanced finding
- life experience → becomes a story seed

### Ask: “How far does this word go?”

Abstract words often have a limited scope.

Examples:

- “universal” may mean everyone under the same conditions, not every person in every possible situation
- “valid” may mean valid for this argument, not universally true
- “important” may mean important to the author’s claim, not important in general

Ask:

- For whom?
- Under what conditions?
- Compared with what?

## 5. Timing system: question-number checkpoints only

Do not calculate passage time. Do not remember passage start times. Do not use timer-wall arithmetic.

Use the countdown clock and the overall question number.

| Countdown timer | Pace checkpoint |
|---:|---|
| 80:00 | Q6 completed / starting Q7 |
| 70:00 | Q12 completed / starting Q13 |
| 60:00 | Q18 completed / starting Q19 |
| 50:00 | Q24 completed / starting Q25 |
| 40:00 | Q30 completed / starting Q31 |
| 30:00 | Q36 completed / starting Q37 |
| 20:00 | Q42 completed / starting Q43 |
| 10:00 | Q48 completed / starting Q49 |
| 0:00 | Q53 completed |

These are approximate checkpoints, not rigid stop commands.

### How to use them

- Check only at the major 10-minute marks.
- Compare the current question number with the chart.
- If one or two questions behind: keep moving; do not panic.
- If several questions behind: stop broad rereads and limit time on 50/50 questions.
- Do not spend time recalculating how much each passage “deserved.”

## 6. Whiteboard setup

The whiteboard is a dashboard, not a notebook.

Write only:

```text
80 Q6 done
70 Q12 done
60 Q18 done
50 Q24 done
40 Q30 done
30 Q36 done
20 Q42 done
10 Q48 done

○ = 50/50
★ = findable later
/ = guessed / no idea

STEM FIRST → JOB FIRST → THEN CHOICES
```

Do not reserve space for full paragraph notes unless a specific accommodation or practice setup proves they are helpful.

## 7. Skipping rules

### Reasonable to return later

- a detail question with a clear searchable phrase
- a “which is not mentioned?” question
- a question where the needed paragraph is clearly identified

### Usually answer before leaving the passage

- inference
- analogy
- author agreement
- function
- main idea
- strengthen/weaken
- assumption
- 50/50 reasoning questions

Reason: once the passage map is cold, rebuilding it costs more time than finishing the reasoning while still inside the passage.

## 8. Answer-change rule

Only change an answer when there is a passage-based reason that can be stated in one sentence.

Do not change because:

- the answer suddenly feels suspicious
- another answer uses a familiar word
- time pressure creates doubt
- the passage is no longer remembered clearly

## 9. Anxiety and attention rules

- A rough first passage is a warm-up, not proof the whole section is failing.
- The first sentence gets one normal read. Do not reread it three times while telling yourself to focus.
- If attention drifts, use one brief physical reset, then return to the sentence. Do not let tapping, hair-picking, or another regulation behavior become the main task.
- When behind, use targeted rereads only. Know exactly what sentence or phrase is being searched for.

## 10. Review template

For each missed or difficult question, record:

1. What was the question asking me to do?
2. What answer did I choose?
3. What relationship did I assume?
4. What relationship did the passage actually support?
5. Which trap was present?
   - right topic, wrong job
   - too broad
   - too narrow
   - true but irrelevant
   - outside knowledge
   - opposite direction
   - half-right
   - detail match that misses the purpose
6. One repair rule:

> Next time, when I see ______, I will ______.

## 11. Current non-negotiables

1. No full-test scan.
2. No passage-time subtraction.
3. Use question-number checkpoints.
4. Read the stem before comparing choices.
5. Name the answer’s job in plain English.
6. Match relationships, not topics.
7. Define abstract words inside the passage’s scope.
8. Return only to genuinely findable questions.
9. Change answers only with evidence.
10. Keep the whiteboard simple.

## 12. Source and version note
