# CARS Thesis Hunt and Answer-Choice Triangulation

## Purpose

This file preserves the CARS strategy that emerged from repeated Jack Westin and AAMC practice, especially the methods that reduced overload, improved passage orientation, and helped convert uncertain comprehension into accurate answers.

The goal is not to memorize every sentence or preview every question. The goal is to create a fast anchor for the passage, preserve the author’s exact relationship between ideas, and prevent stress from turning uncertainty into aimless rereading.

## Core Discovery

**Answer choices can reveal the passage’s testing center without revealing the correct answer.**

When the passage thesis is unclear, scan one answer-choice set that looks global or thesis-shaped. Do not read every question. Do not solve the question yet. Look only for repeated:

- subjects
- verbs
- objects
- relationships
- opposing thesis lanes

Use those patterns to form a rough expectation of what the passage is centrally discussing.

### SVO Thesis Triangulation

- **S — Subject:** Who or what keeps appearing?
- **V — Verb:** What action, judgment, or relationship keeps appearing?
- **O — Object:** What result, target, value, or consequence keeps appearing?

Then form a rough anchor:

> The passage is probably about how **S** does **V** to **O**.

This anchor is provisional. **The passage remains the final authority.** Wrong choices often contain the correct topic but distort the relationship.

## Thesis Hunt Protocol

Use this as a guiding start when test stress makes it hard to remember to think.

1. Read the title.
2. Glance at the answer choices of Question 1 only if the set looks thesis-shaped or globally framed.
3. Do not read all question stems or preview the entire test.
4. Extract repeated subjects, verbs, objects, and relationships.
5. Read one anchor sentence only:
   - a key sentence in paragraph 2, or
   - the last sentence of the second-to-last paragraph, or
   - the conclusion sentence if it is immediately understandable.
6. Begin reading from the start.
7. Let the passage confirm, narrow, or overturn the provisional anchor.

### Time Guardrail

If the thesis scan is not useful within roughly 10–15 seconds, stop and read normally. The strategy should reduce stress, not become a new ritual.

## What Counts as a Thesis-Shaped Answer Set

A thesis-shaped set usually presents competing big-picture claims such as:

- what the author ultimately argues
- what the author would oppose or endorse globally
- what a central concept means across the passage
- which relationship best describes the passage’s argument
- which interpretation the reader should leave with

A set is less useful for thesis triangulation if it consists mostly of:

- dates
- names
- narrow paragraph details
- vocabulary definitions
- isolated examples
- specific retrieval facts

## Reading Strategy After the Thesis Hunt

### Read Linearly, Monitor Structurally

Do not pre-read every first sentence. Do not scan the entire passage first. Read in order, but after each paragraph ask:

> What did this paragraph do?

Use one short job label:

- setup
- common view
- author correction
- claim
- evidence
- example
- mechanism
- contrast
- consequence
- counterargument
- final lesson

The goal is location memory, not perfect wording memory.

### Read the Spine Before the Ribs

For syntax-heavy sentences, identify:

> subject → verb → main claim

Temporarily skip decorative interruptions, then return only if they change the logic or a question requires them.

Usually safe to skip first pass:

- credentials and titles
- name stacks
- location/date piles
- appositives
- parenthetical decoration
- long lists of examples
- stylistic elaboration

Do not skip:

- however, but, yet, although
- rather, instead, not X but Y
- therefore, thus, hence, so
- definitions
- author judgment words
- the sentence that interprets a quotation
- a sentence that states a consequence or recommendation

## Thesis Marination Strategy

If the passage thesis does not fully click after reading, do not force the main-idea question immediately.

1. Mark or skip the global thesis question.
2. Answer one or more concrete questions first:
   - detail
   - paragraph-based inference
   - phrase meaning
   - local function
3. Let those questions clarify which relationships the test considers important.
4. Return to the thesis question once the passage has had time to “marinate.”

This is not permission to let question explanations replace the passage. It is a way to allow local reasoning to sharpen the global picture.

## Question-Stem Compression

Question stems are commands wearing extra language. Strip them to task + target.

- **According to** → retrieve
- **Suggests / implies** → smallest supported step
- **Function / role** → what job does this part perform?
- **Assumes** → what must be true?
- **Most weakens / challenges** → what damages the author’s reasoning?
- **Most supports** → what strengthens the relationship?
- **Based mainly on** → what evidence does the heavy lifting?
- **If true** → pretend each option is true; assess its effect
- **EXCEPT / NOT / LEAST** → reverse the polarity before solving

## Answer-Choice Translation

When two choices remain, translate each into plain subject–verb–object language.

Ask:

1. Which answer preserves the author’s exact relationship?
2. Which answer is narrower and less dramatic?
3. Which answer avoids converting possibility into certainty?
4. Which answer avoids changing “can” into “should,” “sometimes” into “always,” or “different” into “better/worse”?
5. Which answer matches the author’s actual mechanism rather than the topic or emotional tone?

### Recurring Trap Pattern

The most common error is **over-upgrading the author’s claim**.

Examples:

- “Different” becomes “more complex.”
- “Prediction is unreliable” becomes “human intelligence is untrustworthy.”
- “Do no harm” becomes “never act.”
- “Mind and body influence one another” becomes “psychological treatment should come first.”
- “Study can support domination” becomes “learning removes fear.”

When uncertain, prefer the boring, narrower answer that preserves the relationship.

## Returning to the Passage

Do not use “never go back” as a rule.

Use:

> Go back only if I know what I am checking and where it is.

Before rereading, identify:

- the target information
- the likely paragraph or sentence

Good reasons to return:

- detail retrieval
- exact phrase meaning
- analogy mechanism
- function of a specific sentence/example
- implication tied to a known location

Bad reasons to return:

- vague anxiety
- general discomfort
- wanting reassurance
- rereading without a target

## Optional Room Model

The room model is optional, not mandatory. Use it only when it gives an immediate orientation.

- **Courtroom:** What is the author proving, defending, accusing, or correcting?
- **Classroom:** What mechanism, definition, or causal chain is being explained?
- **Theater:** What perspective, technique, meaning, or experience is being interpreted?
- **Newsroom:** What event or trend is being reported, and what angle matters?
- **Conference Room:** What problem, tradeoff, or response is being evaluated?
- **Family Room:** What human, cultural, or moral reflection is being explored?

If classification takes effort, abandon it and return to the direct question:

> What is the author trying to make me see differently?

## Minimal Whiteboard Use

The whiteboard should not become a passage map.

Use it only for:

- skipped question numbers
- timing checkpoints when needed
- one emergency reminder such as `TARGET + LOCATION`

Example:

```text
Skip: 4, 9
Return only with target + location
```

## Anti-Hamster-Wheel Rule

Do not take another passage merely to prove a result.

- A poor score can trigger: “I need to prove I am not bad.”
- A strong score can trigger: “I need to prove it was not a fluke.”

Both reactions create the same unproductive loop.

After a strong or weak set:

1. Record what was tested.
2. Record the strategy result.
3. Identify one keep/modify/stop decision.
4. End the CARS block when the planned number of passages is complete.

Success is evidence. It does not require immediate replication to count.

## Current Testing Protocol

1. Read the title.
2. Use a brief answer-choice thesis scan only if it provides a calm anchor.
3. Read one anchor sentence at most.
4. Read the passage linearly.
5. Track paragraph jobs mentally.
6. Read logic turns and author judgments; skip decorative syntax first pass.
7. If the thesis is still fuzzy, answer concrete questions and return later.
8. Decode each stem into task + target.
9. Translate close answer choices into plain SVO relationships.
10. Go back only with a target and location.
11. Stop after the planned CARS workload; do not chase proof.

> Use answer-choice patterns to locate the battlefield, use the passage to determine the winner, and preserve the author’s exact relationship between ideas.
