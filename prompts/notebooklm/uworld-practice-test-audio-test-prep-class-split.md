# NotebookLM Prompt Set — UWorld Practice Test Audio: Test Prep Class Lecture Mode

Use these prompts when the goal is a full test-prep class lecture, not a short review. These are designed for UWorld MCAT practice tests and should be used as separate NotebookLM audios to avoid shallow coverage.

Core philosophy: no stone unturned. The speakers should read and explain the test like a class instructor: passages, discrete questions, experiments, figures, data, structures, answer choices, equations, wording, traps, and the underlying Kaplan/UWorld content connections.

---

## Prompt 1 — Discrete Questions Only: Test Prep Class Mode

You are creating an MCAT UWorld test-prep class audio for a student.

ONLY cover the discrete questions. Ignore all passage-based questions.

This is not a summary. This is a full test-prep class lecture. Go through the discrete questions slowly and thoroughly, with no stone unturned.

For every discrete question:

1. Read or clearly restate the full question stem.
2. Explain every important term, phrase, symbol, equation, structure, molecule, unit, or wording choice.
3. Translate the question into plain language: what is it actually asking?
4. Identify the relevant Kaplan/UWorld content area or chapter connection when possible.
5. Teach the background concept as if giving a mini-lecture.
6. Explain what the student should recognize immediately.
7. Explain what information is useful and what is distraction.
8. If an equation appears, explain each variable, units, proportional relationships, and how to set it up.
9. If a structure, molecule, reaction, graph, figure, or table appears, explain how to read it from the beginning.
10. Read or clearly restate every answer choice.
11. Explain what each answer choice means.
12. Explain why each answer choice is tempting or wrong.
13. Explain why the correct answer is correct.
14. Identify whether the likely issue is content knowledge, question wording, answer-choice wording, graph/figure interpretation, equation setup, units, math, or reasoning.
15. End with a “next-time move” for this question type.

The audio should feel like a high-quality MCAT prep class where the instructor is walking the student through the test in real time.

Use two tutor-like speakers. They should be calm, detailed, and practical. They should not skip steps. They should not assume the student understands the question just because the content is familiar.

Slow down especially for:
- scientific notation
- unit conversions
- equations
- C/P math
- graph reading
- figure interpretation
- experimental design
- redox
- thermodynamics
- kinetics
- gases
- spectroscopy
- separations and purification
- formal charge
- amino acids and biomolecules
- metabolism and enzymes

End the audio with:
- recurring content gaps
- recurring wording traps
- recurring math/graph/figure traps
- 3–5 rules to carry into the next UWorld block

Do not compress the review to make it shorter. Prioritize complete teaching.

## Prompt 2 — Passages 1–5 + Questions: Test Prep Class Mode

You are creating an MCAT UWorld test-prep class audio for a student.

ONLY cover passages 1–5 and all associated questions. Ignore passages 6–10 and ignore discrete questions.

This is not a summary. This is a full test-prep class lecture. Read and explain the test as if teaching a live MCAT prep class. No stone should be left unturned.

For each passage:

1. Read or clearly restate the passage in order.
2. Explain the title if present.
3. Explain every paragraph slowly.
4. For each sentence or major phrase, explain what it means and why it may matter.
5. Define important terms, abbreviations, variables, molecules, structures, pathways, methods, and experimental labels.
6. Explain the experiment or study design from the beginning.
7. Identify independent variables, dependent variables, controls, comparisons, conclusions, and limitations.
8. If data, figures, graphs, tables, molecules, structures, pathways, reaction schemes, or equations appear, walk through them from the beginning.
9. Explain axes, labels, units, trends, controls, comparisons, and what the student should notice.
10. Connect the passage to relevant Kaplan/UWorld content areas or chapter themes when possible.
11. Teach the needed background content as a mini-lecture, but keep returning to the passage.
12. Explain what looked important but was actually distraction.
13. Explain what the MCAT writer expected the student to notice.

After the passage walkthrough, go through every associated question:

1. Read or clearly restate the full question stem.
2. Translate it into plain language: what is it actually asking?
3. Identify the question type: passage reasoning, figure interpretation, graph interpretation, experiment design, equation use, content recall, pseudo-discrete, or trap question.
4. Point back to the exact passage clue, figure, graph, data point, or concept needed.
5. Read or clearly restate every answer choice.
6. Explain what each answer choice means.
7. Explain why each answer choice is tempting or wrong.
8. Explain why the correct answer is correct.
9. Identify the likely mistake type: content gap, passage misunderstanding, wording misunderstanding, answer-choice misunderstanding, visual misunderstanding, equation setup, unit issue, math issue, or reasoning issue.
10. End with a “next-time move.”

The audio should feel like the student is sitting in a rigorous MCAT test-prep class where the teacher reads the passage, decodes the experiment, teaches the underlying science, and then teaches every question and answer choice.

Use two tutor-like speakers. They should be detailed, calm, and practical. They should not skip terms, sentence meanings, answer choices, figures, graphs, or equations.

Use phrases such as:
- “Pause here.”
- “This sentence is doing work.”
- “This is the experiment setup.”
- “This wording is dangerous.”
- “This is where students misread the passage.”
- “This answer is tempting because…”
- “The next-time move is…”

End the audio with:
- recurring science connections from passages 1–5
- recurring experimental reasoning traps
- recurring figure/graph/data traps
- 3–5 rules to use on the next UWorld block

Do not compress the review to make it shorter. Prioritize complete teaching.
