## 2. Current CARS operating statement

> Read for the author's machine. Track where ideas live. Skip clutter, not logic. Verify wording only when you know where to look. When stuck, choose the narrower answer that preserves the author's actual claim.

### Non-negotiables

- **Skimming as the primary method is banned.**
- Strategic **hop-and-return is allowed**.
- Returning must begin with the passage anchor, not with the answer choices.
- Preserve actor → action → target relationships.
- Never skip contrast, causality, concession, or evaluation words.
- Give the final paragraph extra attention because it often resolves the thesis.
- Reread only with a target and probable location.
- Do not preview the entire test.

## 3. Paragraph-job reading

After each paragraph, retain a one-to-three-word job label:

- introduces issue
- old view
- author challenge
- evidence
- example
- concession
- consequence
- proposed solution
- final judgment

The tag is not a summary of every detail. It records the paragraph's function in the author's argument.

### Three-layer engine

| Layer | Purpose | Current use |
|---|---|---|
| Core flow | Claims and logic that must be understood | Read fully |
| Skip-first-pass detail | Names, parenthetical details, dense examples | Move through without losing main syntax |
| Return-if-asked | Specific evidence and definitions | Locate only for targeted retrieval |

Voice layer:

- author thesis or judgment
- neutral narration
- author paraphrase of another view
- direct quotation
- hard evidence

Landmarks:

- main claim
- contrast
- cause/effect
- example

The colors and symbols are teaching tools for review, not a requirement to mark every live passage.

## 4. Skipping versus skimming

The user's successful strategy is **controlled skipping**, not skimming.

Controlled skipping means:

- read the core grammatical clause
- temporarily set aside interruptions such as long appositives or examples
- preserve logical connectors
- know where the skipped detail lives
- return only if the question requires it

Skimming was harmful because it caused loss of:

- actor
- action
- target
- negation
- comparison direction
- author attitude

“I do not remember seeing it” is not evidence that the passage did not say it.

## 5. Hop-and-return

Hop-and-return is useful when a question freezes momentum, but it creates one specific risk: the original passage anchor can be lost.

### Return protocol

    return to question
    → restate the original passage pattern
    → locate the relevant paragraph job
    → read the decisive sentence fully
    → compare the last two answers

Do not re-enter through the answer choices because they can overwrite the passage's original logic.

For **least expected / most expected** questions:

1. State the established pattern first.
2. Ask which choice most clearly follows or breaks that pattern.
3. Then reread the relevant proof.

Example retained from the “Square One” review:

- The anchor was educational entertainment for elementary-school children.
- Advanced calculus broke the established pattern.

## 6. Answer-choice thesis triangulation

The user discovered that answer choices can help reveal the passage's repeated subject-action-object relationship.

Allowed use:

- selectively glance at choices for a main-idea-style question
- identify repeated nouns, verbs, and relationships
- use them as a hypothesis for what AAMC is testing
- confirm against the passage

Not allowed:

- preview every question stem
- let answer choices replace the passage
- select an answer solely because it appears repeatedly
- scatter attention across the whole set

Passage text remains the final authority.

## 7. Question-stem scope control

Translate the stem into its job before evaluating choices.

| Stem cue | Required job |
|---|---|
| Why / because | Explain cause or purpose |
| Central idea | Preserve the passage-wide controlling relationship |
| Function of example | Explain why the author included it |
| According to passage | Retrieve passage-supported claim |
| Inference | Smallest conclusion required by evidence |
| Strengthen/weaken | Change support for the specified claim |
| Suppose / new information | Apply the new condition without pretending it was in the passage |
| NOT / LEAST | Reverse the selection criterion before comparing |
| Most expected / least expected | Preserve or violate the established pattern |

Common error patterns:

- giving a true detail when the stem asks **why**
- giving a broad philosophical summary when the stem asks for a narrow textual claim
- softening a strong author claim
- upgrading “different” into “more complex” or “better”
- choosing the pleasant interpretation rather than the supported one
- answering from outside knowledge

## 8. Grammar outranks unreliable punctuation

The user correctly noticed that copied, exported, OCR, or even test-presented punctuation may fail to clarify sentence structure.

Recovery order:

1. Find the subject.
2. Find the main verb.
3. Find the complement or object.
4. Preserve parallel constructions.
5. Reattach interruptions.
6. Use punctuation as secondary evidence.

Key example:

    not a popular but a literary idea

Structure:

    not X but Y

Meaning:

- It was not a popular idea.
- It was a literary idea.

Do not doubt a sound grammatical parse merely because the commas feel missing.

## 9. Proper nouns, quotations, and highlighting

Proper nouns and quotations are retrieval landmarks, not automatically important ideas.

Selective highlighting may include:

- names likely to be referenced later
- quoted or italicized terms
- explicit thesis language
- contrast or causal hinges
- a definition likely to be retrieved

Avoid:

- highlighting most sentences
- using color as a substitute for comprehension
- highlighting every proper noun with equal weight
- searching the entire passage without a location hypothesis

## 11. Science-passage fear versus actual task

The user repeatedly described scientific passages as visually unfamiliar, jargon-heavy, and intimidating. The main repair is to translate the passage into a small experiment model.

### Passage skeleton

    scientific question
    → system or model
    → manipulation
    → measured outcome
    → result
    → supported conclusion

For each paragraph, ask:

- What is known?
- What changes?
- What is measured?
- What direction does it move?
- What mechanism does the author propose?

Unknown protein names are often labels, not new content that must be memorized.

## 12. Experimental-design map

| Element | Question to ask |
|---|---|
| Independent variable | What did the researchers manipulate? |
| Dependent variable | What did they measure? |
| Control | What comparison isolates the manipulation? |
| Constant | What was held the same? |
| Confounder | What alternative variable could explain both sides? |
| Result | What actually changed, including direction and magnitude? |
| Conclusion | What is supported, without overstating causation? |

### Genetic-manipulation distinctions

- Knockout removes or disrupts function.
- Knockdown reduces expression.
- Knock-in inserts a sequence or specific alteration.
- Overexpression increases gene product.

### Assay rule

Do not memorize only the assay name. Track:

    input
    → biological interaction
    → signal or separation principle
    → readout
    → inference

## 13. Graph and table interpretation

### Required sequence

1. Read the title.
2. Read both axes and units.
3. Identify groups/conditions.
4. State the trend in plain language.
5. Check error bars and significance notation.
6. Only then interpret mechanism.

### Directionality sentence

Use:

> When **X increases/decreases**, **Y increases/decreases** relative to **control**, under **condition Z**.

This prevents:

- reversing axes
- confusing release with binding
- confusing expression with activity
- treating a correlation as causation
- assuming visible separation means statistical significance

### Common Test 5 P/S correction

A graph can show different means without proving statistical significance. Check the study's reported statistics or uncertainty display before making a significance claim.

## 14. Discrete versus passage-integrated questions

| Question type | First move |
|---|---|
| Discrete | Retrieve the content rule; ignore invented passage context |
| Passage retrieval | Locate the cited result or definition |
| Passage application | Combine content rule with passage condition |
| Experimental inference | Use manipulation/result logic |
| Calculation | Translate quantities and units before arithmetic |

The user's Anki strength can create a trap: a familiar card may trigger an answer before the passage's modified condition is incorporated.

## 15. Calculation and unit protocol

### Noun-unit-equation sequence

1. Name the requested quantity.
2. Write its required unit.
3. List given values with units.
4. Choose an equation that cancels to the required unit.
5. Estimate sign and order of magnitude.
6. Calculate.
7. Compare with the estimate.

### Personalized arithmetic support

The user has a long-standing difficulty with mental combinations involving 6, 7, and 8.

Whiteboard repair:

- include a small mixed-number addition box for recurring combinations
- omit same-number facts
- use it as a speed aid, not evidence of content weakness

### Scientific notation

Keep separate:

- coefficient arithmetic
- exponent arithmetic
- unit conversion

Planck's exponent and similar constants should be retrieved through repeated mixed prompts, not only rereading.

## 16. Scratch-paper efficiency

Useful marks:

- arrows for direction
- plus/minus for increase/decrease
- IV, DV, control
- units
- one-line formula setup
- question numbers to return to

Time-wasting marks:

- rewriting the passage
- copying full equations already visible
- drawing elaborate diagrams before identifying the question
- repeatedly recalculating timing checkpoints
- writing explanations that belong in review, not during the test

## 19. Strict autopsy format

Every question review uses:

### What Happened

The observable action, not a personality judgment.

### Why

The causal error: knowledge, reading, retrieval, scope, math, or timing.

### Fix Rule

A short executable rule.

### Trigger

What future wording or visual feature should activate the rule.

### Strategy Status

- Keep
- Modify
- Drop
- Test again

Example:

    What happened: Reversed which molecule was produced.
    Why: Read the pathway nouns but did not draw direction.
    Fix rule: Draw substrate → product before viewing choices.
    Trigger: Any “produces,” “releases,” or “converted to” stem.
    Status: Keep.

## 20. Review-completion criteria

A file, transcript, export, or explanation existing does not mean the material is reviewed.

Count a question as repaired only when the user can:

1. state the original error pattern
2. answer again without viewing the solution
3. explain why the correct answer works
4. reject the tempting wrong answer
5. reproduce the result under timing
6. pass a delayed retrieval check

Transcription is evidence. It is not mastery by itself.

## 22. Mind-map trigger rule

Create or expand a map only when:

- the same concept is missed twice
- a relationship still cannot be explained after reading
- multiple nearby topics blend together
- equation/unit setup repeatedly fails
- a pathway or compartment cannot be tracked
- CARS passage structure repeatedly collapses

Do not create a new mind map for every one-off missed fact.

Mind maps organize relationships. They do not replace retrieval practice or timed application.

## 23. Test-day visual and physical setup

### Zoom and highlighting

- Use a zoomed-out level that keeps more passage visible and reduces scrolling.
- Lean forward if needed rather than constantly changing zoom.
- Highlight selectively; do not use heavy highlighting as the only mental map.

### Breaks

- Use brief, structured physiological resets.
- Avoid phone/social disengagement that destroys reading momentum.
- Water, snack, standing/stretching, and whiteboard stabilization take priority.

### Distraction controls

Prepare known stress-picking triggers before the exam:

- braid or secure hair
- trim nails and remove hangnails
- exfoliate lips
- consider sleeves
- pluck chin hairs beforehand
- check chair/armrest comfort when possible

For the erasable noteboard:

- follow testing-center rules
- use thumb/sleeve only if allowed and marker is wet-erase
- if marker behaves as permanent, cross out rather than attempting removal

## 24. Personalized reasoning error engine

Before choosing an answer:

1. What exact job does the stem assign?
2. What is the passage's controlling relationship?
3. Where does the evidence live?
4. Am I preserving actor, action, target, direction, and qualifier?
5. Is this content, passage application, experiment, graph, or calculation?
6. Does my answer say more than the evidence?
7. If returning, did I restore the original anchor first?

Highest-priority repairs:

- **No primary skimming.**
- **Skip interruptions, never logic.**
- **Return through the passage anchor.**
- **Grammar structure outranks confusing punctuation.**
- **Pick the narrower supported answer.**
- **Translate the stem before judging choices.**
- **Read axes and units before interpreting a graph.**
- **Name IV, DV, control, and result.**
- **Use units to choose equations.**
- **A review is complete only after independent, delayed retrieval.**
