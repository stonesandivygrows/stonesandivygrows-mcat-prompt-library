## How To Continue

1. Read the workflow rules and this handoff first.
2. Read the tracker to identify the next eligible topic.
3. Fetch/read any existing target Whimsical board before planning. If the tracker says `new-board-needed`, create a new topic board only after approval.
4. Work by topic, not broad subject.
5. Prefer existing Anki-repackaged boards when they substantially cover the topic.
6. Avoid Periodic Table, nucleic/DNA/RNA, and interdisciplinary boards unless the user explicitly reopens them.
7. Make one concise plan and wait for explicit `yes` before board work.
8. After board work, fetch/read back the board and verify intended diagrams exist.
9. Update only concise GitHub metadata: tracker row, update-log entry, and this handoff if the resume state changes.

Style rules:

- Max 3-4 child branches per node.
- Do not put 5+ direct child nodes under one parent.
- Use logical intermediate groupings for concepts with many details.
- Prefer depth over breadth.
- Use a compact main hierarchy map plus focused flowcharts, tables, and 1-2 confusion/caution notes where helpful.
- Keep all board content study-facing. Do not put workflow/process notes inside boards.
